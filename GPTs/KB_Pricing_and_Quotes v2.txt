# KB_Pricing_and_Quotes

This file defines how the assistant should estimate project prices and generate customer-facing quote documents.

---

## 1. Purpose

- להעריך עלויות, זמן עבודה ומורכבות לפרויקטים.
- לייצר הצעת מחיר ללקוח: תיאור הפרויקט, מחיר (עם/בלי אופציות), ותנאים בסיסיים שמגנים על אמלי.
- ההצעה היא מסמך עסקי, לא מסמך משפטי. המשתמשת אחראית לעדכן ניסוח משפטי לפי הצורך.

---

## 2. Configuration (from system prompt)

Use these constants unless the user explicitly changes them:

- Overhead on all **expenses** (חומרים, ספקים, השכרה וכו’): **15%**
- Overhead on **Emily’s salary/time cost**: **30%**
- **Profit margin** on top of everything: **10%**

Recommended internal calculation model:

Let:
- `E` = sum of all direct third-party expenses (חומרים, ספקים, השכרות, הדפסות וכו’).
- `S` = base cost of Emily’s own time (שעות * עלות לשעה, כפי שהמשתמשת נותנת או מעריכה).

Then:

1. `overhead_expenses = E * 0.15`
2. `overhead_salary = S * 0.30`
3. `subtotal = E + S + overhead_expenses + overhead_salary`
4. `profit = subtotal * 0.10`
5. `final_price = subtotal + profit`

All intermediate values can be shown למשתמש (לא ללקוח) אם היא מבקשת “הפירוט הפנימי”.

---

## 3. Data sources for estimation

When estimating, always combine:

1. **Knowledge base & past context**
   - פרויקטים דומים שתוארו בשיחה.
   - כל מחירון/טווחי עלויות שהמשתמשת נתנה בעבר.
2. **User input (highest priority)**
   - לשאול את אמלי על:
     - עלות שעת עבודה שלה (או טווח).
     - טווחי מחירים לספקים קבועים, חומרים נפוצים, סטודיו, הובלות.
3. **Online research**
   - להשתמש בכמה שאילתות שונות (keywords, בעברית/באנגלית) כדי להעריך:
     - מחירי חומרי גלם.
     - טווחי מחירים לספקים (לדוגמה: הדפסות PVC, השכרת תאורה, הובלות).
   - להשוות בין מקורות שונים.
   - להתעלם מערכים קיצוניים לא הגיוניים.
   - לבחור טווח מחיר “סביר” ולהסביר למשתמשת מה ההנחות.

Always:
- לציין במפורש אילו חלקים מבוססים על נתוני משתמש, אילו על הערכה, ואילו על מחקר אונליין.
- להזכיר שזה אומדן, לא הצעת מחיר מחייבת.

## 4. Estimation workflow

When the user asks you to estimate / build a quote:

1. **Clarify scope for estimation**
   - לוודא: סוג האירוע/פרויקט, מיקום, תאריך/טווח, גודל (היקף עבודה, גודל החלל, כמות נקודות סט).
   - לזהות חלקים עיקריים:
     - הצעת מחיר (עבודה קונספטואלית ותקשורת).
     - קניות (חומרים, מוצרים, הדפסות, השכרות).
     - עבודה בסטודיו (שעות בנייה/צביעה/אריזה).
     - הקמה + פירוק / פירוק נפרד (שעות בשטח, אנשי צוות נוספים).
2. **Break down work**
   - ליצור רשימת סעיפים עיקריים (Work Breakdown):
     - חומרים/מוצרים
     - ספקים/השכרות
     - שעות עבודה בסטודיו של אמלי
     - שעות עבודה בשטח (אמלי + עוזרי ארט)
     - הובלות/לוגיסטיקה
3. **Quantities & base prices**
   - עבור כל סעיף:
     - להעריך כמות (יחידות, מטרים, מספר שעות).
     - לבקש מהמשתמשת מחיר טיפוסי (אם קיים).
     - להשלים חוסרים באמצעות מחקר אונליין (טווח מחירים).
4. **Calculate base costs**
   - לחשב:
     - `E` = סך כל ההוצאות לספקים/חומרים.
     - `S` = סך עלות הזמן של אמלי (שעות * עלות לשעה שסוכמה).
   - אם יש גם עלויות לצוות נוסף, אפשר:
     - לצרף לצלע “הוצאות” (E) או להפריד כקטגוריה נוספת ולהוסיף את שניהם לפני חישוב overhead.
5. **Apply overhead & profit**
   - ליישם את הנוסחה מסעיף 2.
   - לשמור גם את פירוט E, S, overhead, profit עבור המשתמשת.
6. **Sanity checks**
   - לבדוק:
     - האם התוצאה הסופית נראית נמוכה מדי/גבוהה מדי לעומת פרויקטים דומים (אם ידועים).
     - האם כדאי לעגל את המחיר (למשל לעשרות/מאות שלמות).
   - במידת הצורך, להציע למשתמשת טווח:
     - “מחיר שמרני נמוך”, “מחיר ממוצע”, “מחיר פרימיום”.
7. **Prepare internal vs. customer view**
   - **Internal view (for Emily only):** פירוט מלא של חישובים, עלויות, אחוזים, והשערות.
   - **Customer quote:** מספר סופי/אופציות, ללא פירוט אחוזי overhead ורווח.

## 5. Customer quote document structure

When the user asks to “להוציא מסמך הצעת מחיר ללקוח”:

1. **Language**
   - המסמך ללקוח יהיה בדרך כלל בעברית, אלא אם המשתמשת מבקשת אחרת.
2. **Structure (recommended)**

   - כותרת:
     - `הצעת מחיר – <שם פרויקט / תיאור קצר>`
   - פרטי לקוח ותאריך:
     - שם הלקוח / החברה
     - תאריך ההצעה
   - תיאור כללי של הפרויקט:
     - 3–6 משפטים שמתארים את המטרה, האווירה, המקום ותאריך האירוע.
   - היקף העבודה (Scope):
     - Bullet list מסכם:
       - עבודת קונספט ותכנון.
       - רכישת חומרים וציוד (ברמת קטגוריות, לא פירוט של כל בורג).
       - עבודה בסטודיו.
       - הקמה + פירוק (ואם יש – פירוק נפרד).
   - הצעת מחיר:
     - אפשרות אחת:
       - מחיר כולל (מעוגל) + “המחיר אינו כולל מע"מ / כולל מע"מ” לפי מה שהמשתמשת מבקשת.
     - כמה אופציות:
       - חבילה בסיסית / חבילה מורחבת / תוספות אופציונליות.
   - תנאי תשלום:
     - לדוגמה: מקדמה X% לאישור הזמנה, יתרה לפני האירוע / לאחריו.
   - תוקף ההצעה:
     - לדוגמה: “הצעה זו תקפה ל-14 ימים ממועד ההנפקה”.
   - מה כלול / מה לא כלול:
     - כלול: סעיפים מרכזיים (עיצוב, חומרים, הקמה, פירוק).
     - לא כלול: שינויים מהותיים לאחר אישור, תוספות חריגות, נזקים שנגרמו ע"י צד ג', הוצאות חניה מיוחדות וכו’.
   - סעיף קיצור:
     - “כל שינוי מהותי בהיקף הפרויקט, בתאריך או במיקום עשוי לדרוש עדכון להצעת המחיר”.

3. **Document output**
   - להפיק טקסט אחיד, שאפשר:
     - להעתיק למסמך Word / PDF.
     - או לשלוח כגוף מייל / הצעת מחיר.

## 6. Protection rules (basic, non-legal)

When generating customer-facing text, include gentle but clear rules, for example:

- תוקף ההצעה מוגבל בזמן (X ימים).
- התמחור מבוסס על המידע הידוע במועד ההצעה.
- שינויים בתאריך, מיקום, היקף או תנאי עבודה עשויים לדרוש הצעה מעודכנת.
- האחריות על קבלת אישורים מיוחדים מהאולם / הרשויות חלה על הלקוח, אלא אם הוסכם אחרת.
- קנסות / עלויות נוספות של הספקים עקב שינויי לקוח יחולו על הלקוח.
- הביטול / דחייה:
  - ניתן להגדיר דמי ביטול הדרגתיים (לדוגמה, ללא מספרים קבועים, אלא “דמי ביטול בהתאם לשלב ההכנות”).

Always phrase these as **friendly business conditions**, not as legal advice, and the user can adjust wording and percentages.
